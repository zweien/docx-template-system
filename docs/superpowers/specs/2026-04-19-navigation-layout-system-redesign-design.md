# 导航布局系统重构设计（基于 DESIGN.md）

- 日期：2026-04-19
- 目标范围：Dashboard 导航布局系统（Sidebar / MobileNav / Header 及导航状态与路由匹配）
- 设计基准：`DESIGN.md`（Linear 风格深色原生体系）

## 1. 背景与目标

当前项目已做多轮 UI 改造，但导航系统仍存在历史问题：

1. 交互可用性问题（折叠、移动端联动、激活态准确性、键盘可访问性）。
2. 视觉一致性问题（局部仍保留旧样式，层级与 token 使用不统一）。
3. 侧边栏相关控制台报错风险（客户端状态读取、监听器与渲染时机）。

本次重构目标：

1. 以 `DESIGN.md` 为统一视觉规范，重构完整导航布局系统。
2. 采用业务域分组并按“执行优先”顺序重排导航。
3. 解决侧边栏历史问题（交互 + 视觉）。
4. 将“导航系统相关”控制台错误收敛到 0。

## 2. 范围与非目标

### 2.1 范围（In Scope）

1. `Sidebar`（桌面导航）
2. `MobileNav`（移动抽屉导航）
3. `Header`（顶部动作区）
4. 导航配置、路由匹配、状态管理与副作用管理
5. 与导航强耦合组件（通知入口、用户入口、主题切换入口）的视觉一致性收口

### 2.2 非目标（Out of Scope）

1. 大规模业务内容区重构（除导航耦合调整外）
2. 非导航模块的控制台错误清理
3. URL 结构与业务路由语义改动

## 3. 信息架构设计

### 3.1 主导航分组与顺序（执行优先 + 业务域）

1. 仪表盘
2. 我要填表
3. 生成记录
4. 我的草稿
5. 模板管理
6. 主数据
7. 文档收集
8. AI 助手

### 3.2 管理区（管理员可见）

1. 系统设置
2. 用户管理
3. 审计日志

### 3.3 文案策略

1. 允许小幅命名优化，但不改变业务语义。
2. 同层级命名保持一致性，避免风格混杂。

## 4. 架构与模块拆分

### 4.1 单一导航源（Single Source of Truth）

新增导航配置模块，桌面与移动共用：

- `src/components/layout/navigation/schema.ts`

字段建议：

- `id`
- `label`
- `href`
- `icon`
- `section`（main/admin）
- `order`
- `roles`
- `featureFlag`（可选）
- `hidden`（可选）

### 4.2 路由匹配模块

新增：

- `src/components/layout/navigation/matcher.ts`

职责：

1. 首页精确匹配。
2. 其余路由边界安全前缀匹配（避免 `/data` 命中 `/datax`）。
3. “最长匹配优先”，确保仅一个主激活项。

### 4.3 状态模块

新增：

- `src/components/layout/navigation/state.ts`

职责：

1. `collapsed`：桌面折叠态（持久化）。
2. `mobileOpen`：移动抽屉开关（路由切换自动关闭，不持久化）。
3. 暴露统一 action：`toggleCollapsed` / `openMobile` / `closeMobile`。

### 4.4 渲染层

1. `Sidebar.tsx`：纯渲染 + 交互绑定，不再内嵌业务判断。
2. `MobileNav.tsx`：复用同一 schema 与 matcher。
3. `Header.tsx`：保持全局动作定位，不承载导航业务逻辑。

## 5. 交互规范

### 5.1 侧边栏

1. 折叠态：仅保留 icon，文本平滑过渡。
2. 展开态：分组清晰，激活项唯一。
3. Tooltip：折叠态显示，移动端禁用。
4. `aria-current="page"`：当前路由项必须设置。

### 5.2 移动端

1. 抽屉与桌面共享导航源。
2. 点击菜单后自动关闭抽屉。
3. 与桌面折叠态互不污染。

### 5.3 键盘与可访问性

1. 导航项可 Tab 访问。
2. 图标按钮必须有 `sr-only` 标签。
3. `Cmd/Ctrl+K` 监听器仅注册一次并正确卸载。

## 6. 视觉规范落地

遵循 `DESIGN.md`：

1. 深色原生背景层级（`#08090a` / `#0f1011` / `#191a1b`）。
2. 边框采用 `rgba(255,255,255,0.05~0.08)`。
3. 激活态使用品牌紫低透明背景 + 细 ring。
4. 文字层级：
   - 主要：`#f7f8f8`
   - 次要：`#d0d6e0`
   - 弱化：`#8a8f98`
5. 字重优先：400 / 510 / 590，避免 700。

## 7. 控制台报错修复清单（仅导航相关）

1. `window/localStorage is not defined`
   - 仅在客户端安全时机读取。
2. hydration mismatch（导航状态导致）
   - 初始渲染使用稳定默认值，挂载后同步持久化状态。
3. 重复事件监听 warning
   - 快捷键监听统一封装并确保 cleanup。
4. 列表 key warning（导航列表）
   - 使用稳定 `id` 作为 key。

## 8. 实施顺序

1. 建立 schema + matcher（不改 UI）
2. 接入 state hook（折叠/移动开关/持久化）
3. 重构 Sidebar
4. 重构 MobileNav
5. 重构 Header 导航耦合逻辑
6. 清理监听器与副作用
7. 视觉对齐与细节收口
8. 回归验证

## 9. 验收标准（DoD）

1. 导航在桌面/移动端行为一致。
2. 折叠状态持久化稳定，移动端抽屉不污染桌面状态。
3. 激活态准确（含嵌套路由）。
4. 导航相关控制台 error 为 0。
5. 视觉符合 `DESIGN.md`。

## 10. 测试计划

### 10.1 功能测试

1. 不同角色菜单可见性正确。
2. 侧边栏折叠/展开刷新后状态正确。
3. 移动端点击导航后抽屉自动关闭。

### 10.2 交互测试

1. 键盘 Tab 遍历导航可用。
2. 当前项 `aria-current` 生效。
3. `Cmd/Ctrl+K` 不重复触发。

### 10.3 稳定性测试

1. 导航相关页面流转时控制台无 error。
2. 无 hydration mismatch。

### 10.4 视觉回归

1. Sidebar/Header/MobileNav 与 `DESIGN.md` 色阶一致。
2. 激活、hover、focus 态一致。
