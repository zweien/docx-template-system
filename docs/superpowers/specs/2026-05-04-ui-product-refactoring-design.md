# UI 产品级重构设计

> 日期：2026-05-04
> 状态：已批准
> 范围：全面产品级重构 — 视觉统一 + 导航重构 + 首页重设计

## 背景与目标

系统前端经过多轮功能迭代（模板管理、数据表、报告撰写、自动化、文档收集、AI 助手等），38 个页面、213 个组件分布在 19 个功能目录。由于各模块独立开发，视觉风格逐渐割裂：

- **卡片容器**：5 种写法（rounded-xl/2xl、bg-card/70/90、shadow inset 等）
- **标题排版**：4 种风格（font-[510]/[520]/bold、tracking-[-0.7px]/[-0.04em]/tight）
- **空状态**：2 种完全不同的实现（div vs Card 容器）
- **侧边栏**：16 项平铺，缺乏层次和分组

**目标**：从完整产品角度重构 UI，统一视觉语言、重组导航结构、重设计首页，参照 DESIGN.md 中的 Linear 风格规范。

## 1. 导航结构重构

### 设计决策

采用**统一模块模式**（飞书多维表格风格），按功能域划分，侧边栏从 16 项精简为 7 个顶层项。

### 侧边栏结构

```
首页

模板与表单 ▾          ← 合并：模板库 + 填表 + 记录 + 草稿
  模板库               /templates
  我要填表             /generate
  生成记录             /records
  我的草稿             /drafts

数据中心 ▾             ← 独立域
  数据表               /data

报告中心 ▾             ← 独立域
  撰写报告             /reports/drafts
  报告模板             /reports/templates
  预算报告             /budget

─────────────────
自动化                 /automations      ← 横切工具，独立项
文档收集               /collections      ← 横切工具，独立项
智能助手               /ai-agent2        ← 横切工具，独立项

─────────────────
管理后台 ▸             ← 二级入口，仅 ADMIN 可见
  系统设置             /admin/settings
  用户管理             /admin/users
  审计日志             /admin/audit-logs
  AI 配置              /admin/editor-ai
```

### 行为规则

- **折叠**：模块组默认展开当前所在组，其余折叠。折叠时只显示组名，点击组名直接进入该组第一个子页面。
- **收起状态**：侧边栏收起时只显示图标 + tooltip，模块组显示组图标，hover 展开子菜单 popover。
- **管理后台**：收为二级入口（点击展开或弹面板），不污染普通用户视野。
- **路由不变**：所有 URL 保持向后兼容，零破坏性变更。

### 侧边栏实现

- 重构 `src/components/layout/sidebar.tsx`
- 更新 `src/components/layout/navigation/schema.ts`，增加 `group` 字段和 `children` 层级
- 新增 `NavGroup` 组件，处理折叠/展开逻辑

## 2. 设计系统统一

严格遵循 DESIGN.md 中的 Linear 风格规范。

### 2.1 色彩体系

| 语义 | 值 | 用途 |
|------|-----|------|
| `--background` | `#08090a` | 页面底色 |
| `--sidebar` | `#0f1011` | 侧边栏背景 |
| `--card` | `#191a1b` | 卡片/面板 |
| `--foreground` | `#f7f8f8` | 主文字 |
| `--muted-foreground` | `#8a8f98` | 次要描述 |
| `--dim-foreground` | `#62666d` | 最弱文字 |
| `--primary` | `#5e6ad2` | 品牌强调 |
| `--accent` | `#7170ff` | 交互状态 |
| `--border` | `rgba(255,255,255,0.08)` | 标准边框 |
| `--border-subtle` | `rgba(255,255,255,0.05)` | 弱分隔 |

**禁止**：魔法值（如 `bg-card/70`、`bg-card/90`、`shadow-[inset_0_0_0_1px_...]`）

### 2.2 字体系统

| 用途 | Tailwind 类 |
|------|------------|
| 页面标题 | `text-2xl font-[510] tracking-tight` |
| 区块标题 | `text-base font-medium` |
| 正文 | `text-sm` |
| 辅助文字 | `text-xs text-muted-foreground` |
| 页面描述 | `text-[15px] text-muted-foreground tracking-tight mt-1` |

**禁止**：`font-[520]`、`font-bold`（用 590）、`tracking-[-0.7px]`、`tracking-[-0.04em]`

### 2.3 圆角系统

| 元素 | 圆角 |
|------|------|
| 卡片/面板/弹窗 | `rounded-lg`（8px） |
| 按钮/输入框 | `rounded-md`（6px） |
| 小元素/Tag/Badge | `rounded`（4px） |
| 筛选标签 | `rounded-full`（9999px） |

**禁止**：`rounded-2xl`、`rounded-xl` 混用

### 2.4 阴影规范

| 层级 | 使用场景 |
|------|---------|
| 无阴影（默认） | 卡片、列表容器 |
| `shadow-sm` | hover 状态、弹出面板 |
| `shadow-lg` | Dialog、Sheet |

**禁止**：自定义 inset shadow、`shadow-[0_12px_30px_...]`

### 2.5 间距规范

| 场景 | 值 |
|------|-----|
| 页面区块间距 | `space-y-6` |
| 卡片网格间距 | `gap-4` |
| 卡片内边距 | `p-6`（标准）/ `p-4`（紧凑） |
| 按钮组间距 | `gap-2` |

## 3. 统一页面布局模式

38 个页面归纳为 4 种布局模式，每种有明确的组件组合公式。

### 3.1 模式 A：列表页（7 个页面）

适用于：模板库、生成记录、数据表列表、自动化列表、文档收集、报告列表、用户管理。

```
<PageHeader title description actions=[搜索, 新建] />
<FilterBar pills=[全部, 已发布, 草稿, 已归档] />
<ContentCard>
  <Table ... />
  <Pagination />
</ContentCard>
```

### 3.2 模式 B：详情页（6 个页面）

适用于：模板详情、记录详情、自动化配置、收集任务详情、报告草稿详情、数据表详情。

```
<Breadcrumbs items=[...] />
<PageHeader title actions=[编辑, 填写] />
<Tabs items=[基本信息, 配置, 历史] />
<ContentCard>
  {/* Tab 内容 */}
</ContentCard>
```

### 3.3 模式 C：表单/向导页（5 个页面）

适用于：填写表单、上传模板、批量生成、导入数据、创建自动化。

```
<Breadcrumbs />
<PageHeader title />
<StepIndicator steps=[...] current={1} />
<ContentCard>
  <FormFields />
</ContentCard>
<ActionBar>
  取消 | 上一步 | 下一步
</ActionBar>
```

### 3.4 模式 D：工作台（1 个页面）

适用于：首页仪表盘。详见第 4 节。

## 4. 首页工作台重设计

从"统计卡片堆砌"升级为"高效工作入口"。

### 设计原则

- **操作优先**：统计数字缩小到底部，快捷操作和待办占据核心位置
- **串联模块**：首页成为所有模块的枢纽
- **轻量统计**：只保留 4 个关键数字

### 布局结构

```
工作台
早上好，{用户名}。有 {n} 项待办任务。

┌─────────────┬─────────────┬─────────────┐
│  我要填表    │  撰写报告    │   数据表     │  ← 快捷操作（3 卡片横排）
│  选择模板..  │  编辑/创建.. │  管理数据..  │
└─────────────┴─────────────┴─────────────┘

┌──────────────────┬──────────────────┐
│  我的待办 (3)     │  最近使用        │  ← 双列
│  ├ 季度考核表     │  ├ 劳动合同 [再填] │
│  ├ 资产登记表     │  ├ 月度报告 [继续] │
│  └ 报销单审批     │  └ 花名册 [查看]   │
└──────────────────┴──────────────────┘

┌───────────────────────┬──────────┐
│  团队动态              │ 本月统计  │  ← 底部双列
│  ├ 王五 生成 出差申请单 │  47 文档  │
│  ├ 李四 完成 保密协议   │  12 待办  │
│  └ 张三 上传 请假条     │  8 模板   │
└───────────────────────┴──────────┘
```

### 数据来源

- **我的待办**：来自文档收集模块中分配给当前用户的未完成任务 + 自动化模块中需要人工审批的节点
- **最近使用**：当前用户最近填写的模板（"再填一次"）、最近编辑的报告（"继续编辑"）、最近查看的数据表
- **团队动态**：最近 10 条生成记录 + 收集任务提交记录，按时间倒序
- **本月统计**：当月生成的文档数、待办任务数、活跃模板数、自动化运行次数

## 5. 新增共享组件

| 组件 | 用途 | 替代 |
|------|------|------|
| `PageHeader` | 统一页面标题+描述+操作按钮 | 各页面手写的 h1 + p + button 组合 |
| `EmptyState` | 统一空状态展示（图标+标题+描述+操作） | div/Card 两种不同空状态实现 |
| `FilterBar` | 统一筛选标签栏（pill 形式） | 各页面不同的筛选栏实现 |
| `ContentCard` | 统一卡片容器（border + bg + radius） | 5 种不同卡片写法 |
| `NavGroup` | 侧边栏分组折叠组件 | 当前的平铺导航项 |
| `Breadcrumbs` | 面包屑导航 | 部分页面已有，统一规范 |

## 6. 实施策略

### 阶段划分

**阶段 1：基础设施（设计系统 + 共享组件）**
- 更新 CSS 变量对齐 DESIGN.md
- 创建 6 个共享组件（PageHeader、EmptyState、FilterBar、ContentCard、NavGroup、Breadcrumbs）
- 影响范围：零功能变更，纯视觉基础

**阶段 2：导航重构**
- 重构侧边栏为分组导航
- 更新 navigation schema
- 折叠/展开逻辑
- 影响范围：所有页面的侧边栏

**阶段 3：首页重设计**
- 实现新首页工作台布局
- 新增待办、最近使用、团队动态等 API
- 影响范围：仅首页

**阶段 4：逐页统一**
- 按页面模式（A/B/C/D）逐页应用共享组件和设计规范
- 优先级：列表页 > 详情页 > 表单/向导页
- 影响范围：所有页面

### 向后兼容

- 所有 URL 路由不变
- 功能逻辑不变
- 仅视觉呈现和导航结构变更
- 阶段 1-2 可独立发布，不影响功能
